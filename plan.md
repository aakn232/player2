## Plan: 윈도우 홈캠 연속 재생 플레이어

1분 단위 MP4를 재인코딩 없이 하나의 가상 타임라인으로 재생하기 위해, 파일 인덱싱 계층과 타임라인 매핑 계층을 먼저 고정하고 그 위에 플레이어 UI를 얹는 방식으로 진행한다. 권장 구현은 Python + PySide6 + libmpv이며, 이유는 윈도우 배포 용이성, 고속 재생 안정성, 긴 재생 시간에서의 리소스 효율성 때문이다.

**Steps**
1. 범위 고정 및 입력 규칙 정의: 단일 루트 폴더에서 날짜 폴더 단위로 MP4를 스캔하고, 파일명에서 시간 정보를 파싱하는 규칙을 확정한다. 파싱 실패 시 타임스탬프 보조 규칙을 둔다.
2. 세그먼트 인덱서 구현 설계: 각 파일을 Segment 메타데이터로 정규화한다. 포함 필드는 경로, 시작시각, 길이, 순번, 무결성 상태이며, 정렬 기준은 실제 시간순으로 고정한다. (선행 단계)
3. 가상 타임라인 엔진 설계: 전체 길이를 계산하고 전역 초 단위 위치를 Segment+오프셋으로 변환하는 탐색 로직을 설계한다. 이진 탐색 기반으로 즉시 점프 성능을 보장한다. (2에 의존)
4. 재생 백엔드 어댑터 설계: libmpv 제어 계층을 두고 파일 로드, 오프셋 시킹, 배속(최대 16x), 일시정지, 다음 세그먼트 선로딩을 제공한다. (3에 의존)
5. 경계 전환 로직 설계: 파일 끝 경계에서 다음 파일로 자연스럽게 넘어가도록 버퍼링 전략을 정의한다. 오디오 끊김과 프레임 드롭 감시 포인트를 포함한다. (4에 의존)
6. UI 설계: 24시간형 탐색 바, 현재 시각 표시, 배속 컨트롤(1x/2x/4x/8x/16x), 날짜/시간 점프 입력을 포함한다. 탐색 바 클릭 시 전역 시간 점프를 수행한다. (3,4와 병행 가능)
7. 오류 처리 및 복구 설계: 누락 파일, 손상 파일, 코덱 불일치, 폴더 순서 오류를 사용자에게 명확히 표시하고 재생 가능한 구간만 계속 재생하는 정책을 정의한다. (2~6과 병행 가능)
8. 윈도우 패키징 계획: PyInstaller 기반 단일 실행 파일 배포, mpv 의존성 번들링, 초기 설정 파일 위치 정책을 확정한다. (4~7에 의존)
9. 성능 검증 계획: 24시간 분량 기준 랜덤 시킹 응답시간, 경계 전환 지연, 16x 연속재생 안정성을 수치로 측정한다. (전체 단계 완료 후)
